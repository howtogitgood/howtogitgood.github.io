How to Get Good with Git

A guide on how to get started with git and version control

By Joe Guidoboni

This guide is intended to be the guide for git that I wish I had when I started as a CS student. I'm going to cover what git is and how to get started.

Git is a Version Control System or VCS. Basically, VCS's store changes made to a file or files. This set of instructions will cover initializing git, adding files, committing changes, and reverting to a previous commit. It assumes that you already have git installed and have a basic understanding of using the command line.

1. Creating a repository

Go to an empty folder that you want to create a git repository or repo in, and type the following into the terminal:

git init

When done, you should see an output like something below:

Initialized empty Git repository in /your/empty/folder/.git/

If you don't see this, then git most likely isn't installed on your computer. You can install git here.

Now files that you put in this folder and add to git will now be tracked by git.

2. Adding files to a repository

Now that we've created a repository, we want to add files that can be tracked by git. To do this, we place the files that we want to keep track of into the folder that the repository is located in, and then type into the Terminal:

git add -A

Where -A is a flag that specifies that you want git to keep track of all of your files. Git will now keep track of all of our files for us and we can edit it in any way we want. To check to see if the files were added properly, type:

git status

You should see something like the following:

On branch master

No commits yet

Changes to be committed:
  (use "git rm --cached <file>..." to unstage)
        new file:   text

If you get something like this instead:

On branch master

No commits yet

nothing to commit (create/copy files and use "git add" to track)

Then the file wasn't properly added. Try typing in the add command again and check to see if the file you want to add is in the folder that you made the repository in.

3. Commits

So far you've created a repo and added a file to it. Now you want to save the changes that you've made to the file using a commit. We have to re-add all the files again here before the commit is made too. Type the following lines into the terminal:

git add -A
git commit -m "This is a message"

Where the section in quotes is a message that describes what you've done to the file since the last commit. Think of commits as checkpoints that make it easier to figure where and when things went wrong.

4. Reverting to Previous Commits

If we want to go back to a previous commit, we have to find the hash value of the commit that we want to go back to. To find the hash value, type the following into the terminal:

git log

This displays all the information about all of your previous commits like this:

commit 40d94e0d65f5e7f9b40d2f3b912f169b2d3b12b1 (HEAD -> master)
Author: Joe Guidoboni <jrg1057@wildcats.unh.edu>
Date:   Tue Mar 24 10:00:46 2020 -0400

    Third commit

commit 42b103a76116d70cd32beb450eea75c68956ee5e
Author: Joe Guidoboni <jrg1057@wildcats.unh.edu>
Date:   Tue Mar 24 09:55:56 2020 -0400

    Second commit

commit 65c166fa56062285cb6f36f6037fa72b83a1f704
Author: Joe Guidoboni <jrg1057@wildcats.unh.edu>
Date:   Tue Mar 24 09:55:14 2020 -0400

    First commit

The big long strings of numbers and letters are called hashes. In this example, we're going to go back to the commit with the message "First Commit", which has the hash value:

65c166fa56062285cb6f36f6037fa72b83a1f704

To revert to a previous commit, find the hash of the commit that you want to go to, and type 

git reset --soft hash-value

where the â€”soft flagmeans that I still have things that I'm working on that I don't want to overwrite when I go back to the previous commit, and the hash-value is the hash of the commit that you want to go back to. Using the example hashvalue, we would type:

git reset --soft 65c166fa56062285cb6f36f6037fa72b83a1f704

To check if it was done correctly, type:

git log

You should see something like the following, where the we are back at the commit that you wanted to revert to:

commit 65c166fa56062285cb6f36f6037fa72b83a1f704 (HEAD -> master)
Author: Joe Guidoboni <jrg1057@wildcats.unh.edu>
Date:   Tue Mar 24 09:55:14 2020 -0400

    First commit

Wrapping up

That's all there is to do basic operations with git. Git is a very robust system and is designed so that it is difficult to mess up the files that you are working on beyond repair.
